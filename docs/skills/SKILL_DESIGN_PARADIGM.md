# 杀戮尖塔 AI Advisor - Skill 设计范式

## 概述

本文档定义 STS AI Advisor 的知识注入范式，采用**两级索引**架构，将攻略文档重新组织为结构化的知识表示。

## 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│  第一级：流派识别层（全量注入）                                   │
│  ─────────────────────────────────────────────────────────────  │
│  体积极小（~50个流派卡片），让 LLM 快速识别当前卡组归属           │
│  输出：匹配到的流派 + 置信度                                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ 识别后按需加载
┌─────────────────────────────────────────────────────────────────┐
│  第二级：详细策略层（按流派索引）                                 │
│  ─────────────────────────────────────────────────────────────  │
│  分层内容：                                                      │
│  - Layer 1: 数值计算（Tool+规则）                                │
│  - Layer 2: 局部策略（启发式规则）                               │
│  - Layer 3: 涌现价值（范例库）                                   │
│  - Layer 4: Roguelike决策（决策树）                              │
└─────────────────────────────────────────────────────────────────┘
```

## 第一级：流派识别层

### 设计原理

- **不切片原文**，而是用 LLM 将攻略压缩为标准化卡片
- 卡片体积极小，可**全量注入** System Prompt
- LLM 通过模式匹配识别当前卡组最接近的流派

### 流派卡片 Schema

```json
{
  "id": "strength-scaling",
  "name": "力量叠加流",
  "character": "ironclad",
  "core_mechanism": "通过力量buff使每张攻击牌的伤害倍增，单卡价值随力量线性增长",
  "essential_cards": ["突破极限", "双持", "恶魔形态"],
  "synergy_cards": ["重刃", "连击", "剑柄打击"],
  "anti_synergy_cards": ["全能", "岿然不动"],
  "formation_signals": [
    "拥有2张以上力量来源",
    "卡组攻击牌平均伤害>8",
    "有重刃/连击等力量乘数牌"
  ],
  "relic_priority": ["工具箱", "陀螺", "日晷"],
  "key_potions": ["力量药水", "液态记忆"],
  "strengths": ["后期爆发极高", "遗物协同强"],
  "weaknesses": ["前期脆弱", "启动慢", "怕多层小怪"],
  "boss_matchup": {
    "favorable": ["六火鬼", "天罚"],
    "unfavorable": ["时间吞噬者", "心脏"]
  },
  "risk_level": "high",
  "skill_path": "skills/ironclad/strength-scaling/SKILL.md"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，用于索引 |
| name | string | 是 | 流派中文名 |
| character | string | 是 | 角色：ironclad/silent/defect/watcher |
| core_mechanism | string | 是 | 一句话描述核心机制 |
| essential_cards | string[] | 是 | 必要牌，缺了不成流派 |
| synergy_cards | string[] | 是 | 协同牌，有了更强 |
| anti_synergy_cards | string[] | 否 | 反协同牌，有了变弱 |
| formation_signals | string[] | 是 | 判断卡组是否属于该流的信号 |
| relic_priority | string[] | 否 | 遗物优先级（从高到低） |
| key_potions | string[] | 否 | 关键药水 |
| strengths | string[] | 是 | 流派优势 |
| weaknesses | string[] | 是 | 流派劣势 |
| boss_matchup | object | 否 | Boss克制关系 |
| risk_level | string | 是 | 风险等级：low/medium/high |
| skill_path | string | 是 | 第二级详细策略路径 |

### 注入方式

流派卡片在 System Prompt 中全量注入，格式如下：

```xml
<archetype_index>
  <archetype id="strength-scaling">
    <name>力量叠加流</name>
    <signals>
      - 拥有2张以上力量来源
      - 有重刃/连击等力量乘数牌
    </signals>
    <essential>突破极限, 双持, 恶魔形态</essential>
  </archetype>
  <archetype id="block-deck">
    <name>防御流</name>
    <signals>
      - 多张高格挡牌
      - 有岿然不动/幽灵铠甲
    </signals>
    <essential>岿然不动, 幽灵铠甲</essential>
  </archetype>
  <!-- ... 更多流派 -->
</archetype_index>
```

### Layer 1: 数值计算层

**范式**: Tool + 验证规则

**内容形式**: 计算公式、边界条件、验证逻辑

```markdown
# Layer 1: 数值计算

## 伤害计算公式

```
最终伤害 = (基础伤害 + 力量) × 攻击次数 × 乘数效果 - 敌人格挡

其中：
- 力量：正数加伤害，负数减伤害
- 攻击次数：连击/双持等
- 乘数效果：脆弱(×1.5)、虚弱(×0.75)
```

## 格挡计算

```
最终格挡 = 基础格挡 + 灵巧 + 格挡加成

注意：
- 灵巧来自遗物/卡牌
- 格挡加成来自:dexterity(灵巧药水)等
```

## 能量效率

```
能量效率 = 总价值 / 消耗能量

高能量效率操作：
- 0费牌：效率无限大
- 过牌：考虑抽到的牌的价值
```

## 边界条件

- 最低伤害: 0（不能为负）
- 最低格挡: 0
- 最大手牌数: 10
- 最大能量: 无上限（但受遗物限制）
```

### Layer 2: 局部策略层

**范式**: 启发式规则

**内容形式**: if-then 规则、优先级列表、决策checklist

```markdown
# Layer 2: 局部策略

## 回合内决策流程

### Step 1: 致死判断（最高优先级）

```
IF 敌人下回合伤害 >= 当前HP - 当前格挡:
    优先级 = "防御"
    计算需要的最小格挡
    选择能提供该格挡的最优牌组合
```

### Step 2: 杀敌判断

```
IF 本回合能打死敌人:
    计算所需伤害
    选择能量效率最高的输出组合
```

### Step 3: 资源优化

```
IF 无致死/杀敌威胁:
    优化能量使用
    考虑：过牌、上Buff、留存关键牌
```

## 出牌优先级规则

### 防御优先规则

| 条件 | 策略 |
|------|------|
| 敌人下回合伤害致死 | 必须防御，计算最小格挡 |
| 敌人下回合伤害>30%HP | 优先防御 |
| 有脆弱debuff | 格挡价值×1.5 |
| 敌人有多段攻击 | 每段都要覆盖格挡 |

### 输出优先规则

| 条件 | 策略 |
|------|------|
| 能量≥3 | 优先高费牌（效率更高） |
| 能量=1 | 优先0费牌 |
| 有力量加成 | 多段攻击牌优先 |
| 敌人有护甲 | 考虑穿透/重攻击 |

### 过牌时机

| 条件 | 是否过牌 |
|------|----------|
| 手牌质量差 + 有过牌 | 是 |
| 手牌已有核心牌 | 否 |
| 能量紧张 | 否 |
| 有"牌位"遗物 | 优先过牌 |

## 状态管理

### Debuff 处理优先级

1. 虚弱 > 脆弱 > 其他（影响生存）
2. 有净化牌时评估价值
3. 考虑敌人debuff持续时间

### Buff 施加时机

- 力量Buff：在输出回合前施加
- 格挡Buff：在防御回合前施加
- 能量Buff：手牌多时使用
```

### Layer 3: 涌现价值层

**范式**: Few-shot 范例库

**内容形式**: 模式描述 + 典型范例 + 价值变化表

```markdown
# Layer 3: 涌现价值

> 核心洞察：单张牌的价值不是固定的，它依赖于卡组的其他组成。

## 卡组模式识别

### 模式定义

本流派（力量叠加流）的核心模式：

**核心引擎**: 力量 × 多段攻击 = 指数级增长

**价值函数**: 攻击牌价值 = 基础伤害 + 力量 × 攻击次数

### 牌价值变化表

| 牌名 | 孤立价值 | 本流派价值 | 变化原因 |
|------|----------|------------|----------|
| 重刃 | ★★☆☆☆ | ★★★★★ | 伤害 = 5 + 力量×5，力量高时爆炸 |
| 连击 | ★★★☆☆ | ★★★★★ | 多段攻击完美乘力量 |
| 突破极限 | ★★☆☆☆ | ★★★★★ | 力量来源核心 |
| 全能 | ★★★★☆ | ★★☆☆☆ | 不吃力量加成 |
| 岿然不动 | ★★★★☆ | ★★☆☆☆ | 资源分散，不如专注力量 |

## 典型范例

### 范例 1：成型力量流

```
卡组（18张）：
  核心：突破极限×2, 双持×1, 重刃×2
  攻击：剑柄打击×2, 连击×1
  防御：幽灵铠甲×2, 火焰护盾×1
  过牌：战吼×2, 燃烧契约×1
  其他：恶魔之火×1, 祭品×1

遗物：工具箱, 陀螺

分析：
- 成型信号：2张突破极限 + 重刃，力量流确认
- 牌位：18张，偏少，过牌够用
- 风险：防御薄弱，需要快速结束战斗
```

### 范例 2：未成型但有潜力

```
卡组（24张）：
  核心：突破极限×1
  攻击：重刃×1, 连击×1, 打击×4
  防御：防御×4, 幽灵铠甲×1
  其他：...

分析：
- 信号：有突破极限+重刃，但只有1张力量来源
- 建议：继续寻找力量牌，暂时按普通流派打
- 选牌：力量牌优先级↑↑
```

### 范例 3：反例 - 不要混流派

```
卡组（22张）：
  力量组件：突破极限×2, 重刃×1
  防御组件：岿然不动×2, 幽灵铠甲×2
  其他：...

问题分析：
- 同时走力量流和防御流，资源分散
- 岿然不动不吃力量，价值下降
- 建议：专注一边，放弃岿然不动或放弃力量
```

## 选牌决策

### 看到这些牌时

| 牌 | 是否选择 | 条件 |
|---|---------|------|
| 突破极限 | 必拿 | 已经决定走力量流 |
| 双持 | 必拿 | 有力量牌时 |
| 重刃 | 优先 | 有力量来源时 |
| 连击 | 优先 | 有力量来源时 |
| 全能 | 不拿 | 已经走力量流 |
| 岿然不动 | 谨慎 | 防御会分散资源 |

### 删除牌优先级

1. 诅咒牌
2. 打击/防御（基础牌）
3. 与流派不兼容的牌
4. 低效率攻击牌
```

### Layer 4: Roguelike决策层

**范式**: 决策树 + 状态机

**内容形式**: 条件分支、概率评估、长远规划

```markdown
# Layer 4: Roguelike 决策

## 路线规划

### 决策树：精英打不打

```
START
  │
  ├─ HP检查
  │   ├─ HP > 60% → 继续
  │   └─ HP ≤ 60% → 不打精英
  │
  ├─ 卡组成型度
  │   ├─ 已有核心轴 → 打（拿遗物）
  │   └─ 未成型 → 继续判断
  │
  ├─ 精英类型判断
  │   ├─ 拉缠 → 风险高，继续评估
  │   ├─ 天罚 → 风险中，继续评估
  │   └─ 三奴隶 → 风险低，倾向打
  │
  └─ 层数判断
      ├─ Act 1 → 精英价值高，倾向打
      ├─ Act 2 → 看卡组
      └─ Act 3 → 看Boss路线
```

### 商店策略

```
商店优先级：
1. 删除牌（卡组>20张时）
2. 核心遗物（流派相关）
3. 核心牌（流派相关）
4. 药水槽
5. 普通遗物
6. 普通牌

删除顺序：
1. 诅咒 > 打击/防御 > 普通攻击
2. 但：循环流需要打击，评估后再删
```

## Boss 战准备

### Time Eater 准备

```
特点：12张牌触发回血+加力量

策略：
- 控制出牌数 ≤ 11/回合
- 爆发流优势（一回合解决）
- 力量流劣势（需要多回合）

准备：
- 选牌时避免太长的combo
- 优先高单卡价值的牌
```

### The Heart 准备

```
特点：
- 第2回合开始每回合打108血
- 有无敌阶段
- 需要持续输出+生存

策略：
- 必须有稳定防御
- 必须有持续输出
- 药水预留关键牌

准备：
- 格挡遗物价值↑
- 力量药水价值↑

## 内容生成流程

### Step 1: 收集原始文档

收集各类攻略文档，来源包括：
- Wiki 文档
- 社区攻略
- 高手复盘
- 速通指南

### Step 2: 生成流派卡片

使用 LLM 批量处理原始文档：

```
系统提示：
你是杀戮尖塔专家，请将以下攻略文档压缩成标准流派卡片。
格式严格按照 JSON Schema，不得包含格式外的信息。

JSON Schema:
{流派卡片schema}

[攻略原文]

输出：JSON格式的流派卡片
```

### Step 3: 人工校验

检查关键字段：
- essential_cards 是否准确
- formation_signals 是否可判断
- boss_matchup 是否合理

### Step 4: 组织第二级内容

根据流派卡片，组织对应的 Layer 1-4 文档。
